## 步驟(依序)

0. **限制讀取範圍**：這個版本不開放廣域 repo 搜尋/巡檢。只讀本任務文字、ref、scope 與其
   直接引用的 source/test；不得全庫列檔、掃 generated/minified assets、node_modules、coverage、
   build、trace 或 video。指定範圍不足以安全完成時執行 issue，不自行擴大搜尋。
1. **收拾現場**:工作區若有未 commit 的殘留(前人 crash 留下的),先判斷:
   - 殘留改動對應得上本任務描述 → 是本任務做到一半,接手做完。
   - 對應不上本任務描述 → 不要用全域 `git reset --hard` 或 `git clean -fd`。只精準還原／移除能由
     diff、任務紀錄與路徑證明是本 track 前輪產生的殘留；無法判定所有權就保留並 issue，避免刪除未知工作。
2. **判斷本任務是否已完成**:對照任務描述與實際程式碼(前人可能已做完,等你驗收)。
   - **任務含明確且仍未解決的 human gate** → 不論完成與否都不實作、不 done,執行
     `<<ISSUE_CMD>> "一句話描述等待的決策"` 回報後結束,不得代替人類決策。
   - **未完成/有缺陷** → 實作或修正;跑 `<<VALIDATE_CMD>>` 直到綠;
     只 commit 屬於本任務的變更(含你依步驟 1 接手的前人殘留),不夾帶無關檔案,
     commit message 建議帶上 <<TASK_TAG>>;工作區收乾淨,不留未 commit 的檔案。
     **本輪有任何 commit 就不必執行 done 命令**——完成確認留給之後的輪次獨立判斷。
   - **已完成** → 什麼檔案都不要動、不要 commit,執行:
     `<<DONE_CMD>>`
     已完成的定義(缺一即為未完成):
     - 任務描述中每條 DoD 指定的命令或檢查,本輪都**實際執行**並取得該 DoD 明定的通過結果;
       只確認「測試檔存在」「程式碼有對到」不算執行,不得只憑印象或前人紀錄認定。
     - 之後再實際跑 `<<VALIDATE_CMD>>` 為綠。
     - 任一結果為失敗、未知或未執行,不得執行 done;任務含未解決的 human gate 時,
       改執行 issue 回報,不得代替人類決策。
     - 跑完用 `git status` 檢查驗證副作用。受版控檔案的變更一律先視為任務尚未完成：依任務、
       diff 與 build 契約判斷它是否是應提交的 production artifact；應提交就完成驗證並 commit，
       所有權或必要性不明就保留並 issue。不得只為製造乾淨現場而 `git restore`。
     - 未追蹤產物只有在 validator/build 契約明確證明是 disposable 且不屬於交付時才可精準移除；
       不得全域 clean，也不得猜測刪除。任何未釐清異動都不能執行 done。
     (task id 必須一字不差;連續多輪 agent 一致同意且無任何異動,任務才算收斂。)

## 非互動與停止規則

- 這是非互動式自動執行環境。不得詢問使用者、等待確認、列出選項或只建議「下一步」。
- 若本輪完成實作並已通過驗證、commit 且工作區乾淨，工作到此完成：**立即停止**；
  不要再執行 `done`，也不要繼續分析、修改檔案、執行其他命令或向使用者提問。
- 若本輪判定任務早已完整，必須實際執行 `done`；命令成功後立即停止。
- 若任務有問題，必須實際執行 `issue`；命令成功後立即停止。issue 會成為下一輪 agent 的
  context，不等於預設等待人工。
- 若執行工具或權限遭拒，明確輸出 `tool permission blocked` 與原始錯誤後立即停止；
  不得假裝命令已執行，也不得要求使用者代為執行。
- 若多次嘗試後本輪仍無法把驗證修綠：不要 commit 紅燈變更、不要硬跑 done；
  輸出目前進度、已嘗試的方向與驗證失敗尾段後直接停止，殘留交由下一輪依「收拾現場」判斷。
  只有任務本身不可行或描述錯誤才改走 issue。

## 禁區

- goal、參考文件、workspace 的 state.json / state.last-good.json / 計畫 JSON 是受保護真相:直接改檔會被偵測、還原、該輪作廢。
- 執行期計畫已凍結:不能新增/修改/跳過任務。若你判斷任務本身有問題(做不了、描述錯),
  執行 `<<ISSUE_CMD>> "一句話描述問題"` 回報後直接結束,把證據交給下一輪 agent 繼續收斂——不要硬做,
  也不要假裝完成。只有需求意圖、安全／不可逆外部狀態或新外部權限的重大未決衝突才是人工 gate。
