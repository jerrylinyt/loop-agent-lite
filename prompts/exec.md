## 步驟(依序)

1. **收拾現場**:工作區若有未 commit 的殘留(前人 crash 留下的),先判斷:
   - 是本任務做到一半 → 接手做完。
   - 是垃圾 → `git checkout .` + `git clean -fd` 清掉。
2. **判斷本任務是否已完成**:對照任務描述與實際程式碼(前人可能已做完,等你驗收)。
   - **未完成/有缺陷** → 實作或修正;跑 `<<VALIDATE_CMD>>` 直到綠;
     只 commit 你本輪動過的檔案,commit message 帶上 <<TASK_ID>>;工作區收乾淨,不留未 commit 的檔案。
     **本輪有任何 commit 就不必執行 done 命令**——完成確認留給之後的輪次獨立判斷。
   - **已完成且無可挑剔** → 什麼檔案都不要動、不要 commit,執行:
     `<<DONE_CMD>>`
     (task id 必須一字不差;連續多輪 agent 一致同意且無任何異動,任務才算收斂。)
